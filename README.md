# For Ponttu

A small personal website — four pages telling your story, built to be pushed straight to GitHub Pages.

## What's inside
- `index.html` — home page
- `story.html` — your timeline
- `gallery.html` — photo gallery (9 photos already added)
- `letter.html` — an interactive letter (tap the envelope to open it)
- `styles.css`, `script.js` — shared styling and interactions
- `images/` — your photos, already resized for the web

## Before you publish
Open `letter.html` in a text editor and personalize the letter further — add the specific thing you want to apologize for or promise to change. The current text is intentionally kept honest but general.

## How to publish with GitHub Pages
1. Create a new repository on GitHub (e.g. `for-ponttu`). Keep it **private** if you don't want it publicly discoverable — GitHub Pages works with private repos too if you're on GitHub Pro, or you can keep it public but give the link only to her.
2. Upload all the files in this folder to the repository (drag-and-drop on github.com works, or use `git`).
3. Go to the repo's **Settings → Pages**.
4. Under "Build and deployment", set Source to **Deploy from a branch**, branch `main`, folder `/ (root)`. Save.
5. Wait a minute, then your site will be live at:
   `https://<your-username>.github.io/<repo-name>/`
6. Share that link with her, whenever you're ready.

## Making changes later
Every page is plain HTML — open any `.html` file in a text editor, edit the text between the tags, save, and re-upload (or `git push`) to update the live site.
